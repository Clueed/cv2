## What this CV deliberately does NOT do

Several moves were available and explicitly not taken:

- **No invented voice-AI / telephony domain expertise**. The CV does not claim AI-telephony, voice-AI, conversational-AI, or DACH-SMB-buyer reps Mark does not have. The *why-fonio, why-voice-AI* answer is reserved for the cover letter and screen — not faked in the CV.
- **No PhantomBuster / Smartlead name-drop**. Both tools are named in the JD; Mark has not used them. Including them would invite a fast-fail on the technical interview (JD's 90-min GTM Case format includes "Technical Interview mit GTM Case"). Tools that *are* used (HubSpot, Apollo, Lusha, Instantly via the email engine, Clay-adjacent ICP work) are named verbatim where they exist; the rest is held.
- **No SMB-volume-motion rebrand**. The Stackgini bullets describe the enterprise/intra-IT motion ([[feedback_stackgini_buying_centers]]) that Mark actually ran. Reframing as SMB volume-outbound on the CV would invite verification questions Mark cannot defensibly answer. The mechanic-translation gap (intra-IT enterprise → SMB volume) is reserved for the cover letter / screen.
- **No "founding GTM Engineer" title-claim**. Mark holds "Founding GTM Lead" — the GTM-Engineer translation is implied by the bullet content, not asserted.
- **No first-90-days proposal on the CV**. Held for the cover letter / screen. Specific candidate beats: ICP-database build for fonio's pan-European SMB ICP; the LLM-API tooling pattern transplanted onto fonio's stack; the deliverability-stack motion at SMB-volume scale.
- **No demand-gen lexicon** ("MQL", "BANT", "demand capture", "funnel"). Held per the JD analysis recommendation ([[references/fonio-gtm-engineer-jd]] §"Application angle if pursued" beat 4) — fonio is an AI-native scaleup, not a horizontal AI lab, and demand-gen jargon would read as wrong-shape for the GTM-Engineer seat.

The choice across these holds: lead on **scale + shipped GTM-engineering builds + JD-tool-vocab alignment**; let the cover letter carry **why-fonio + DACH-SMB-voice-AI thesis + intra-IT-to-SMB-motion translation + progression question**.
